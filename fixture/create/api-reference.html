Indexing

Functions for creating and maintaining the in-memory search index.

createSearchIndex()

Creates and returns a new, empty SearchIndex. The index holds three complementary lookup structures — headingIds (exact heading text → records), headingIndex (prefix map), and bodyIndex (body word → records) — plus a headingTrie for efficient autocomplete, per-field BM25F frequency tables, and a popularity ring-buffer.

ParamTypeDefaultDescription
No parameters.

Returns: SearchIndex

const index = createSearchIndex();
// index.allSections === []
// index.totalDocs === 0

addToIndex(index, records, pipeline?)

Adds an array of IndexRecord objects to the index. Populates headingIds, headingIndex, headingTrie, bodyIndex, per-field BM25F frequency tables (fieldDocFreq), and allSections.

ParamTypeDefaultDescription
indexSearchIndex—Target index.
recordsIndexRecord[]—Records to add.
pipelineTokenFilter[]undefined (uses built-in stop-word/diacritic/stem pipeline)Custom tokenization pipeline.

Returns: void

addToIndex(index, [
  { id: '/page#intro', url: '/page#intro', headingText: 'Introduction',
    headingId: 'intro', breadcrumb: 'Docs', bodyText: 'Getting started…', type: 'section' }
]);
// Custom pipeline example:
const pipeline: TokenFilter[] = [
  tok => tok === 'the' ? null : tok,          // custom stop-word
  tok => tok.replace(/tion$/, 't'),           // custom stem
];
addToIndex(index, records, pipeline);

addSectionsToIndex(index, records, pipeline?)

Alias for addToIndex — identical behaviour, kept for backward compatibility.

removeFromIndex(index, id)

Removes a record by its id. Also performs finer-grained query-cache invalidation: only cached queries whose terms overlap with the removed record's tokens are evicted; unrelated cached queries are preserved.

ParamTypeDescription
indexSearchIndexTarget index.
idstringThe IndexRecord.id to remove.

Returns: void

removeFromIndex(index, '/page#intro');

updateRecord(index, record)

Replaces an existing record in-place. Internally calls removeFromIndex then addToIndex; cache invalidation covers both old and new tokens.

ParamTypeDescription
indexSearchIndexTarget index.
recordIndexRecordUpdated record (must have same id).

Returns: void

updateRecord(index, { ...existingRecord, bodyText: 'Updated content…' });

getAllSections(index)

Returns the flat allSections array — all records regardless of type.

Returns: IndexRecord[]

serializeIndex(index) / deserializeIndex(json)

serializeIndex converts the index to a JSON string. The trie is not serialized — it is rebuilt on deserialization. deserializeIndex parses that JSON and reconstructs all Maps and the trie, with an automatic migration path for old indices that lack fieldDocFreq (frequencies are approximated from docFrequency).

const json = serializeIndex(index);
// Store: localStorage.setItem('reef', json);
// or pass to saveIndex() in cache.ts

const restored = deserializeIndex(json);
// trie is rebuilt, fieldDocFreq migrated if absent

Searching

searchSections(query, index, limitOrOptions?)

The core search function. Accepts a plain number limit (legacy) or a full SearchOptions object. Returns IndexRecord[] unless includeScore or includeMatches is set, in which case it returns ScoredRecord[].

Scoring pipeline (in order):

  1. Exact heading match → 100 × fieldWeight × (1 + lengthNorm)
  2. Prefix heading match → 50 × fieldWeight × (1 + lengthNorm)
  3. Body-word matches → 20 × fieldWeight × (1 + lengthNorm)
  4. BM25F full scan (only when scoringAlgorithm: 'bm25f' and no hits from above)
  5. Proximity bonus multiplier for multi-term queries (max 2× for adjacent terms)
  6. Exact-phrase bonus: +50% for contiguous token sequence
  7. Popularity multiplicative boost (when trackPopularity: true)
  8. Staged fuzzy fallback (1-typo then 2-typo)
  9. Extended query syntax fallback (when extended: true)
  10. MMR re-ranking (when diversify: true)
OptionTypeDefaultDescription
limitnumber8Max results returned.
fuzzybooleanfalseEnable Levenshtein-based typo tolerance (staged: 1-edit then 2-edit).
fuzzyDistancenumber2Max edit distance for fuzzy stage 3.
includeScorebooleanfalseReturn ScoredRecord[] with normalized 0–1 score.
includeMatchesbooleanfalseInclude MatchSpan[] positions for highlight.
scoringAlgorithm'reef-classic' | 'bm25' | 'bm25f''reef-classic'Ranking algorithm. See Scoring guide.
bm25fOptions{ k1?: number; b?: number }{ k1:1.5, b:0.75 }BM25F saturation (k1) and length-norm (b) tuning.
weightsPartial<Record<'headingText'|'bodyText'|'label'|'breadcrumb', number>>{ headingText:2, bodyText:1, label:1.5, breadcrumb:0.5 }Per-field score multipliers.
extendedbooleanfalseEnable extended query syntax (!exclude, 'phrase', etc.).
filter(record: IndexRecord) => boolean—Post-score filter predicate.
sortFn(a,b: ScoredRecord) => number—Custom comparator applied after default score sort.
typeWeightsPartial<Record<IndexRecord['type'], number>>—Multiplicative boosts per record type.
diversifybooleanfalseApply MMR re-ranking for result diversity.
mmrLambdanumber0.5MMR diversity weight. 0 = pure diversity, 1 = pure relevance.
trackPopularitybooleanfalseApply query-popularity ranking boost.
popularityBoostnumber1.2Base multiplicative popularity boost factor.
 
// Basic search
const results = searchSections('install', index, 10);

// BM25F with tuning
const bm25fResults = searchSections('install', index, {
  scoringAlgorithm: 'bm25f',
  bm25fOptions: { k1: 1.2, b: 0.5 },
  limit: 10,
  includeScore: true,
});

// Diversified results
const diverse = searchSections('config', index, {
  diversify: true,
  mmrLambda: 0.7,   // 70% diversity weight
  limit: 8,
});

// With popularity feedback
const popular = searchSections('login', index, {
  trackPopularity: true,
  popularityBoost: 1.5,
  limit: 8,
});

parseExtendedQuery(query)

Parses a query string using the extended syntax into a QueryNode AST. Pass the result to matchExtendedNode internally via searchSections({ extended: true }). See Query Syntax guide.

Returns: QueryNode — a recursive tree of { type: 'and'|'or'|'term'|'exact'|'exclude'|'prefix'|'suffix' }.

getSnippet(record, matches, contextChars?)

Generates an HTML snippet with the best match highlighted using <mark> tags. Selects the longest MatchSpan, expands a context window of contextChars characters on each side, and adds … ellipsis when the snippet is not at the document boundary.

ParamTypeDefaultDescription
recordIndexRecord—Source record (uses bodyText, falls back to headingText).
matchesMatchSpan[]—Positions from includeMatches: true.
contextCharsnumber60Characters of context around the match.

Returns: string — HTML with <mark>…</mark> around the best match.

const results = searchSections('install', index, {
  includeMatches: true,
  limit: 5,
}) as ScoredRecord[];

for (const { record, matches } of results) {
  const snippet = getSnippet(record, matches ?? [], 80);
  container.innerHTML += `<p>${snippet}</p>`;
}

levenshteinDistance(a, b) / findClosestWord(query, index, maxDistance?)

levenshteinDistance computes the edit distance between two strings using a space-optimised DP (O(n) space). findClosestWord scans all heading and body-index keys and returns the closest one within maxDistance edits (default 2), or null.

levenshteinDistance('install', 'instal'); // 1
findClosestWord('instalaton', index);     // 'installation'

Analytics & Autocomplete

suggest(query, index, limit?)

Returns up to limit heading-text suggestions for the given prefix. Uses the trie as the primary source (O(prefix-length) traversal), falls back to the prefix-map and then substring-contains matching for older indices or when the trie yields fewer than limit results. Requires at least 2 characters.

ParamTypeDefault
querystring—
indexSearchIndex—
limitnumber10

Returns: string[]

const completions = suggest('inst', index, 5);
// ['Installation', 'Instructions', 'Instance Management']

facets(index)

Counts records by type. Useful for building filter UI (e.g., "Sections (42) | Actions (8) | Files (3)").

Returns: Record<IndexRecord['type'], number>

facets(index);
// { section: 120, action: 15, field: 8, link: 30, file: 4, media: 2, structured: 5 }

trackQuery(index, query, selectedId?) / getPopularQueries(index, n?)

trackQuery appends the query to a ring-buffer (max 100). When selectedId is provided, it also increments a PopularityEntry for the (query, recordId) pair. This data is consumed by the trackPopularity scoring option in searchSections.

getPopularQueries tallies the ring-buffer and returns the top n queries sorted by frequency.

// Track a search + click event
trackQuery(index, 'authentication', clickedRecord.id);

// Later, get popular queries for a trending-searches widget
const trending = getPopularQueries(index, 5);
// ['authentication', 'install', 'config', …]

Caching (cache.ts)

IndexedDB persistence with optional gzip compression and TTL expiry. Exported from cache.ts (not re-exported via search.ts; import directly).

saveIndex(index, metadata)

Serializes the index to JSON, optionally compresses with gzip (via the CompressionStream API when available), and stores in IndexedDB under the reef-index database. Falls back to uncompressed storage on unsupported browsers.

ParamTypeDescription
indexSearchIndexThe in-memory index.
metadataCacheMetadata{ versionHash, buildTime, pageMetadata }.

Returns: Promise<void>

loadIndex(ttl?)

Loads the most recent cached entry. Decompresses if needed. Returns null if no entry exists or if the entry is older than ttl milliseconds. Gracefully handles decompression failures by attempting an uncompressed fallback read.

ParamTypeDefault
ttlnumberundefined (no expiry)

Returns: Promise<{ index: SearchIndex; metadata: CacheMetadata } | null>

import { saveIndex, loadIndex, clearCache } from './src/cache.js';

const cached = await loadIndex(24 * 60 * 60 * 1000); // 24 h TTL
if (cached) {
  useIndex(cached.index);
} else {
  const fresh = await buildIndex();
  await saveIndex(fresh, {
    versionHash: 'abc123',
    buildTime: Date.now(),
    pageMetadata: {}
  });
}

clearCache()

Clears all entries from the reef-index IndexedDB object store.

Returns: Promise<void>

Crawling (indexer.ts — Indexer class)

The Indexer class (in src/indexing/indexer.ts) is used internally by reef.ts and is not re-exported via search.ts. Key methods documented here for reference.

new Indexer(config)

Accepts a ReefConfig object. Initialises an empty index, a robotsCache Map, and a lastCrawlTime counter.

indexer.boot(onReady, showToast?)

Orchestrates the full boot sequence:

  1. If prebuiltIndexUrl is set, fetches and deserializes it.
  2. Otherwise, checks IndexedDB cache (honoring ttl).
  3. If stale/missing, calls fetchSitemapUrls() to discover pages.
  4. Runs fetchPagesParallel() (6 concurrent) with incremental-crawl ETag/hash checks.
  5. Saves the new index + metadata to cache.
  6. Falls back to crawlSameOrigin() if no sitemap URLs are found.

Returns: Promise<void>

indexer.fetchSitemapUrls()

Recursively processes sitemaps. Detects <sitemapindex> documents and follows child <sitemap><loc> entries. Deduplicates via a seenUrls Set.

Returns: Promise<string[]>

// Chunk long articles for better BM25F precision
import { chunkBodyText } from './src/search.js';
const chunks = chunkBodyText(longBodyText, 500, baseRecord);
addToIndex(index, chunks);

// Normalize a URL (strips UTMs, resolves canonical)
import { normalizeUrl } from './src/search.js';
normalizeUrl('https://example.com/page/?utm_source=twitter', pageHtml);
// → 'https://example.com/page'  (or canonical from <link rel="canonical">)

Agent / Workflow

DOM automation via the Agent class (agent.ts) and declarative YAML workflows (workflow.ts). See Workflow guide for the YAML format.

Agent class methods

MethodSignatureDescription
click(selector: string | IndexRecord) → Promise<this>Dispatches a bubbling MouseEvent('click') to the resolved element. Chainable.
type(selector, value) → Promise<this>Sets value on an input/textarea/select and dispatches input + change events (React-compatible via descriptor set).
navigate(url) → Promise<this>Sets window.location.href. Only absolute or root-relative URLs.
submit(selector?) → Promise<this>Dispatches submit on a form or clicks a submit button. Falls back to document-level form/button[type=submit].
extract(selector) → Promise<any>Returns .value for inputs or .textContent for other elements.
back() → Promise<this>Calls window.history.back().
forward() → Promise<this>Calls window.history.forward().
wait(timeout?: number) → Promise<this>Waits timeout ms (default 1000).
findActionable(text) → Promise<IndexRecord | null>Searches allSections for action/field records whose heading or label includes text.
executeWorkflow(steps, options?) → Promise<void>Runs a WorkflowStep[] with retry and error-hook support.
getSession() → AgentSessionReturns current URL, timestamp, cookies, and localStorage snapshot.

Workflow functions (workflow.ts)

FunctionDescription
parseYAML(yaml)Parses a YAML string into a WorkflowDefinition (name + steps + options). Simple line-based parser — no external dependency.
validateWorkflow(def)Returns an array of validation error strings (empty = valid). Checks that required fields are present for each action type.
executeWorkflow(definition, agent)Validates then calls agent.executeWorkflow(steps, options).
 
import { Agent } from './src/agent.js';
import { parseYAML, executeWorkflow } from './src/workflow.js';

const agent = new Agent(index, inspector);

// Chained API
await agent.navigate('/login')
  .then(a => a.type('#email', 'user@example.com'))
  .then(a => a.type('#password', 'secret'))
  .then(a => a.submit());

// YAML workflow
const yaml = `
- action: navigate
  url: /login
- action: type
  selector: "#email"
  value: user@example.com
- action: submit
  selector: form
`;
const def = parseYAML(yaml);
await executeWorkflow(def, agent);
